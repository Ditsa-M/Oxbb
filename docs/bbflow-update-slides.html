oxBB update report

From an existing flow baseline to a BBFlow-style oxDNA model

A concise report on the model conversion, data assumptions, validation path, and repository updates now in Oxbb.

conditional flow matching oxDNA endpoint denoising a1/a3 orientation contract

Model target

Generate relaxed oxDNA conformations from an initial or unrelaxed structure while preserving nucleotide positions, base identities, masks, and orientation axes.

The red arrows represent projected body axes, not decoration: valid a1/a3 vectors are part of the generated state.

starting point

The baseline was useful, but not enough for oxDNA generation.

Existing flow baseline

  • Flow model ideas were available: noise, interpolation time, denoising, and sampling.
  • The baseline did not encode the oxDNA state contract directly.
  • Protein-frame assumptions were avoided instead of being copied into this repo.
  • The initial structure needed to become a first-class conditioning input.

BBFlow-style direction

  • Use a conditional prior that starts near the unrelaxed oxDNA structure.
  • Train an endpoint denoiser to predict the clean relaxed target.
  • Sample with a rectified-flow Euler update toward predicted endpoints.
  • Keep positions and orientations coupled throughout training and sampling.

implementation path

The model was refactored around a conditional endpoint prediction workflow.

1. Build a conditional prior

Gaussian position noise and random orientation vectors are blended toward the initial oxDNA structure with `conditional_prior_gamma`.

2. Corrupt toward the relaxed target

A flow time `t` interpolates prior state to target state for positions and projected a1/a3 orientation vectors.

3. Predict the clean endpoint

The denoiser takes current state, initial state, base embeddings, masks, and time embedding, then outputs target positions and orientations.

4. Sample with Euler updates

The sampler repeatedly predicts the endpoint and advances the current state while re-projecting orientations after every step.

flow equation

The conversion is visible in the state transition.

prior z0 = noise mixed with initial structure
+
time zt = (1 - t) z0 + t xtarget
->
denoiser f(zt, initial, bases, t) predicts xtarget
sample_prior(batch) corrupt_batch(batch, t) OxdnaEndpointDenoiser(...) loss = position + orientation + backbone bond terms sample(): state += dt * (predicted_endpoint - state) / (1 - t)

oxDNA-specific contract

Orientations were made explicit instead of treating structure as positions only.

  • Each nucleotide carries `a1` and `a3` body axes.
  • Generated axes are projected to unit length and mutual orthogonality.
  • A right-handed orientation frame is rebuilt from projected axes.
  • Projection happens after raw model output and after sampling updates.

Core helpers

project_a1_a3(a1, a3) frame_from_a1_a3(a1, a3) a1_a3_from_frame(orientation) orientation_quality(a1, a3, mask)

This keeps downstream oxDNA analysis from receiving arbitrary six-dimensional orientation vectors.

model architecture

The denoiser is compact, plain PyTorch, and trainable in the current repo.

  • Base identities are embedded as categorical inputs.
  • Flow time uses sinusoidal embedding plus an MLP.
  • Input features include current state, conditioning state, base embedding, mask, and optional self-conditioning.
  • A Transformer encoder predicts endpoint positions and raw orientation vectors.
  • Orientation output is projected before returning `a1`, `a3`, and frame tensors.

current status

The result is a usable oxBB baseline, not a full final production model.

  • The current model is intentionally BBFlow-style, not a vendored protein BBFlow stack.
  • It prioritizes short duplexes and hairpins before full-origami scaling.
  • Energy evaluation is optional and does not block metric reporting.
  • The next strongest checks are smoke training on a filtered small zarr set and sample evaluation against held-out trajectories.
